# Responsive Layout System

You may be able to achieve fluid page layouts by simply leveraging the abstracted `flexbox` and `grid` APIs.
You will notice that pages using Elements with `nve-layout` will result in layouts that already grown and shrink with respect to the browser size.

However, we introduced an addition to the layout system that uses [CSS Container Queries](https://developer.mozilla.org/en-US/docs/Web/CSS/@container)
under the hood for true next-gen responsive design!

<nve-alert><nve-icon slot="icon">🎓</nve-icon> Learn more about container queries in this Google <a href="https://web.dev/articles/new-responsive#using_container_queries_today" target="_blank" nve-text="link">web.dev</a> video and article.</nve-alert>

This addition to the layout API gives you the ability to apply conditional layout styling based on the width of the parent element containing an element
with `nve-layout` attributes. Currently it supports `gap`, `padding`, and `hide` across a set of 5 contianer size breakpoints.

The set of container `breakpoint-sizes` are defined as:

- `xs < 150px`
- `sm >= 9150px`
- `md >= 300px`
- `lg >= 600px`
- `xl >= 900px`

The `@breakpoint-size:...` API is introduced where the breakpoint size is added before the spacing value.

For example, you can add conitional based gap sizing using `nve-layout="row gap@xs:xxxs gap@md:md gap@lg:xxxl"`.
The size value after the `:` is the standard size value for the [spacing](./?path=/docs/foundations-layout-documentation--docs#layout-gap-spacing)/[padding](./?path=/docs/foundations-layout-documentation--docs#layout-padding) system.

```html
<div>
  <section class="layout-example" nve-layout="row gap@xs:xxxs gap@sm:xxs gap@md:md gap@lg:xl gap@xl:xxxl">
    <nve-card></nve-card>
    <nve-card></nve-card>
    <nve-card></nve-card>
    <nve-card></nve-card>
    <nve-card nve-layout="hide@sm"></nve-card>
  </section>
</div>
```

<nve-alert status="warning">
  Warning: Notice the inclusion of an extra `div` element wrapping the `nve-layout` element. <br/>
  This is to ensure that the parent element that receives the container query is the size of the elements size and not a grid/page.
 </nve-alert>

## Responsive Gap Breakpoints

The following container query breakpoints are available for gap, replace `...` with one of the 9 gap [spacing](./?path=/docs/foundations-layout-documentation--docs#layout-gap-spacing) values:

- `gap@xs:...`
- `gap@sm:...`
- `gap@md:...`
- `gap@lg:...`
- `gap@xl:...`

## Responsive Padding Breakpoints

The following container query breakpoints are available for padding, replace `...` with one of the 9 padding [padding](./?path=/docs/foundations-layout-documentation--docs#layout-padding) values:

- `pad@xs:...`
- `pad@sm:...`
- `pad@md:...`
- `pad@lg:...`
- `pad@xl:...`

<Canvas of={PadResponsive} />

## Breakpoints for Hiding Elements

The following container query breakpoints are available for hiding:

- `hide@xs`
- `hide@sm`
- `hide@md`
- `hide@lg`
- `hide@xl`

Note that elments will hide at all breakpoints equal to and smaller then the specefied size to begin hiding at.
