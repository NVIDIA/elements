# Stepper

<element-metrics tag="mlv-stepper"></element-metrics>

By showing less information at a time, multi-step workflows allow users to focus better on the content pertinent to each step and decrease the chance of errors.

The Stepper component is composed of a `<mlv-stepper>` parent element and multiple `<mlv-stepper-item>` elements slotted as children. <br />
The `selected` attribute applied by the host application will give the selected step the proper selected styles.

The `disabled` attribute can be applied to an item to get the proper visual cues and prevent interaction. <br />
All default interaction, hover and active states are handled by the elements.

Additionally, all keyboard navigation and [accessibility](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/)
concerns are handled out of the box. <br />
Try using left/right arrow keys on horizontal stepper or up/down on vertical stepper.

## Installation

```typescript
import '@elements/elements/stepper/define.js';
```

```html
<mlv-stepper behavior-select>
  <mlv-stepper-item status="success">Step 1</mlv-stepper-item>
  <mlv-stepper-item status="danger">Step 2</mlv-stepper-item>
  <mlv-stepper-item selected>Step 3</mlv-stepper-item>
  <mlv-stepper-item status="pending">Step 4</mlv-stepper-item>
  <mlv-stepper-item disabled>Disabled</mlv-stepper-item>
</mlv-stepper>
```

## Stateless Stepper

By default Stepper will be stateless. This means selection behavior will have to be handled by the host application. <br />
We have provided a way to opt in to stateful selection behavior where click events and keyboard enter will trigger the selection state.

In all other examples on this page `behavior-select` is set on the parent `<mlv-stepper>` to opt into stateful behavior. The following example
shows the default stateless behavior, where the host app will have to set/remove the `selected` attribute on child `<mlv-stepper-item>`.
