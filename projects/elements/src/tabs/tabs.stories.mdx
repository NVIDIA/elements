# Tabs

<element-metrics tag="mlv-tabs"></element-metrics>

Tabs provide a selection UX, typically used for swapping content shown on a page,
or within a navigation context.

The Tabs component is composed of a `<mlv-tabs>` parent element and multiple `<mlv-tabs-item>` elements slotted as children. <br />
The `selected` attribute when applied to an item by the host application will give the selected tab the proper selected styles.

The `disabled` attribute can be applied to an item to get the proper visual cues and prevent interaction. <br />
All default interaction, hover and active states are handled by the elements.

Additionally, all keyboard navigation and [accessibility](https://www.w3.org/WAI/ARIA/apg/patterns/tabs/)
concerns are handled out of the box. <br />
Try using left/right arrow keys on horizontal tabs or up/down on vertical tabs.


<LinkedHeader>Installation</LinkedHeader>

```typescript
import '@elements/elements/tabs/define.js';
```

```html
<mlv-tabs>
  <mlv-tabs-item selected>Tab 1</mlv-tabs-item>
  <mlv-tabs-item>Tab 2</mlv-tabs-item>
  <mlv-tabs-item>Tab 3</mlv-tabs-item>
  <mlv-tabs-item>Tab 3</mlv-tabs-item>
  <mlv-tabs-item disabled>disabled</mlv-tabs-item>
</mlv-tabs>
```

<LinkedHeader>Stateless Tabs</LinkedHeader>

By default Tabs will be stateless. This means selection behavior will have to be handled by the host application. <br />
We have provided a way to opt in to stateful selection behavior where click events and keyboard enter will trigger the selection state.

In all other examples on this page `behavior-select` is set on the parent `<mlv-tabs>` to opt into stateful behavior. The following example
shows the default stateless behavior, where the host app will have to set/remove the `selected` attribute on child `<mlv-tabs-item>`.
