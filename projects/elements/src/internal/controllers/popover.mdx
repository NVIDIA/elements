# Popovers

Elements provides a set of popover type components based on the popover spec and aria guidelines.

- [popover spec](https://open-ui.org/components/popup.research.explainer)
- [w3c aria patterns](https://www.w3.org/WAI/ARIA/apg/patterns/)

## Available Elements

- [tooltip](?path=/docs/elements-tooltip-documentation--docs): contextual text only hints
- [toggletip](#): contextual interactive hints (in development)
- [dropdown](?path=/docs/elements-dropdown-documentation--docs): interactive navigation or form content
- [dialog](?path=/docs/elements-dialog-documentation--docs): large interactive content, interrupts user flow
- [toast](?path=/docs/elements-toast-documentation--docs): contextual notification
- [notification](?path=/docs/elements-notification-documentation--docs): async non contextual notification
- [drawer](?path=/docs/elements-drawer-documentation--docs): interactive navigation or additional contextual content

## Usage

Popover types are visible by default and can be controlled via the `hidden` attribute or by appending and removing from the DOM.

```html
<nve-tooltip anchor="action-btn">hello there</nve-tooltip>
<nve-button id="action-btn">button</nve-button>
```

## Events

All popovers are stateless and do not auto close. Popovers provide a `close` event to notify when
a user has interacted with the popover in a way that should dismiss the popover. Events do not mutate or alter
the visiblity of the element. This ensures maximum flexiblity, allowing events to be used for running
application specific logic before showing or hidding the element.

Some popover types such as tooltip have complex interactions such as showing when hovered or focused.
To simplify these interactions all popover type elements provide a `open` and `close` event to notify
when the popover is attempting to close or open based ona user interaction.

<Story of={TooltipInteractive} inline={false} height={280} />

```html
<nve-tooltip anchor="action-btn" trigger="action-btn" hidden>hello there</nve-tooltip>
<nve-button id="action-btn">button</nve-button>
<script type="module">
  const tooltip = document.querySelector('nve-tooltip[anchor="action-btn"]');
  tooltip.addEventListener('close', () => tooltip.hidden = true);
  tooltip.addEventListener('open', () => tooltip.hidden = false);
</script>
```

## Behavior Trigger

If your use case does not need to control the popover state you can use the `behavior-trigger` property to
enable automatic state management of the `hidden` state based on the trigger.

<Story of={TooltipBehaviorTrigger} inline={false} height={280} />

```html
<nve-tooltip behavior-trigger trigger="action-btn" hidden>hello there</nve-tooltip>
<nve-button id="action-btn">button</nve-button>
```

## Anchor

The `anchor` property defines what element the popover should position itself relative to. For popovers
like modals this defaults to the document body. The `trigger` property is an optional property that defines
what element triggers the `open` event.

<Canvas of={Trigger} story={{ inline: false, height: 270 }} />

For elements like tooltips this is often the same as anchor however the anchor and trigger are not always
the same as UX patterns such as guided tours may have trigger elements that are not the anchored position.
Both `anchor` and `trigger` can take a string idref or a DOM Element reference.
