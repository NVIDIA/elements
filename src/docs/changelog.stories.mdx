# Changelog

## Pending Release
- Fix keypress acitve state on buttons to only trigger with spacebar and enter keys
- Fix DOM based non-HTML5 validation for better framework support (Angular, Vue, React, etc)

## v0.4.0
### Features
- Added T-Shirt size padding to layout system
- Allow anchors to alternatively wrap mlv-button/mlv-icon-button instances as a fallback

### Documentation
- Added Iconography section to Foundations and re-works Storybook navigation
- Added Documentation on Layout System Padding

### Fixes
- Fix interaction button token contrast levels for light theme
- Fix horizontal alignments on control groups
- Fix missing font weight on control group labels
- Fix missing build output for alert element
- Fix vertical alignment for inline type controls
- Fix label align start for textarea/multi select
- Fix shadow tokens
- Fix dark theme border contrast
- Fix overflow shadow cutoff bug
- Fix extra control gap space for non label
- Fix space key support with button key/click
- Fix token values for shadow/shell
- Add `csso` for css minification
- Cleanup/align statuses in preparation for additional status components like toasts/snackbars


## v0.3.2

### Features
- Added Grid Support to Layout System

### Fixes
- Fixed issue when element placed in shadow root would incorrectly render native chromium icons for form controls
- Fixed issue around layout of the default `mlv-card` content. A `mlv-card-content` sub-component was added.

## v0.3.1

### Fixes
- `mlv-icon-button` co-dependency will automatically register when using isolated imports of form elements
- Update typography system to support `rem` font size properties

## v0.3.0

### Features
- Added Horizontal and Vertical Layout System
- Added initial support for native form elements (input, textarea, select, checkbox, radio, range)

### Documentation
- Added `alpha`, `beta`, `stable` tags/info to stories.

### Fixes
- `mlv-button` correctly triggers one event per submit

## v0.2.1

### Features
- Now supports individual element imports for better performance
  ```javascript
  import '@elements/elements/button/define.js';
  ```
  <br />
- Added `mlv-button-icon` element
  ```html
  <mlv-icon-button interaction="emphasize" icon-name="analytics"></mlv-icon-button>
  ```
  <br />
- Build artifact redesigned to reduce side effects in build
- Added MDX docs to Storybook
- Added Lit testing utils for element unit testing
- Updated Storybook to version 6.5
- `mlv-button` and `mlv-button-icon` now based on base button class w/ submit and aria support
    - Refactored with button behavior Lit controllers
- Added beta release of design token integration into elements

### Fixes
- `mlv-button` and `mlv-button-icon` now support full HTML Form submit events
- `mlv-button` and `mlv-button-icon` now support keyboard interactions (space/enter)

### Breaking Changes
- Button `variant` has been replaced with button `interaction` to align with design
- Removed `icon` property on `mlv-icon`
- Refactored `mlv-card` to a composition / slot based API for more flexibility
    - Adds `mlv-card-header`
    - Adds `mlv-card-footer`

## v0.1.3
- Initial release of alpha version to artifactory
- POC version of Button/Icon/Card elements based in Lit/Vite/Storybook build tooling
